# Screenshots — drop them here

The README under [Demo](../../README.md#demo) reserves slots for these
captures. Naming and placement matter — once a file lands at one of the
paths below, the README auto-renders it.

| Suggested filename | What to capture |
|---|---|
| `dashboard_main.png` | Home Assistant main dashboard view |
| `dashboard_services.png` | HA "Services" panel (ambilight controls, scenes) |
| `dashboard_network.png` | HA "Network" panel (ping sensors, Pi-hole stats) |
| `ambilight_sync.gif` | Short clip: TV color shifts → Hue lamps + PC LEDs follow |
| `firetv_before.png` | Stock Fire TV launcher with Amazon ads visible |
| `firetv_after.png` | Debloated Fire TV launcher (e.g. WolfLauncher) |
| `setup_overview.jpg` | Wide phone shot of the whole living-room setup |

Once added, edit the [Demo section](../../README.md#demo) to inline them:

```markdown
![Main dashboard](docs/screenshots/dashboard_main.png)
![Ambilight in action](docs/screenshots/ambilight_sync.gif)
```

Keep individual files under ~2 MB so the repo stays light. For GIFs over
that, prefer a YouTube/Vimeo link in the README.
